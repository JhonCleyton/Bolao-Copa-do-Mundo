Visão Geral

O Bolão Copa 2026 é uma plataforma web completa para gerenciamento de bolão da Copa do Mundo FIFA 2026 (48 times, 104 partidas). Usuários cadastrados fazem palpites nos jogos, acumulam pontos e competem em rankings por rodada e geral.

🎯
Palpites
1 palpite por jogo, prazo automático
🏆
Rankings
Por rodada e geral
💰
Premiação
R$ 100/inscrição + R$ 10/rodada
📱
Notificações
Email + WhatsApp automáticos

Sistema de Pontuação

O sistema usa pontuação acumulativa: cada componente do palpite é avaliado independentemente.

+2
Acertar vencedor ou empate
+2
Acertar gols do Time A
+2
Acertar gols do Time B
+2
Bônus placar exato completo
Máximo: 8 pontos por jogo

Exemplos

ResultadoPalpiteVencedorGols AGols BExatoTotal
2 × 12 × 1✅ +2✅ +2✅ +2✅ +28
2 × 13 × 1✅ +2❌✅ +2❌4
2 × 11 × 0✅ +2❌❌❌2
1 × 12 × 2✅ +2❌❌❌2
2 × 10 × 2❌❌❌❌0
Na fase mata-mata, a pontuação é calculada somente pelo placar dos 90 minutos. O resultado de prorrogação e pênaltis não conta para pontos — apenas para avançar de fase.

Prazo para Palpites

Por padrão, palpites são bloqueados 10 minutos antes do início de cada partida. O admin pode configurar um prazo personalizado por partida via PUT /api/admin/matches/{match_id}/deadline.

Fases do Torneio (Copa 2026)

A Copa 2026 tem 48 times divididos em 12 grupos de 4. O sistema registra cada fase como stage no model Match:

Valor no DBNomePartidasDescrição
group_stageFase de Grupos7212 grupos × 6 partidas cada
round_of_3216 Avos de Final1632 times (1ºs, 2ºs, melhores 3ºs)
round_of_16Oitavas de Final816 times
quarter_finalQuartas de Final48 times
semi_finalSemifinais24 times
third_placeDisputa 3º Lugar1Perdedores das semis
finalFinal1Os 2 melhores
Mata-mata com empate: o campo penalty_winner ("A" ou "B") deve ser preenchido pelo admin ao finalizar um jogo empatado. Isso determina o avanço de fase sem afetar a pontuação dos palpites.

Migrações de Banco

O sistema usa migrações leves e automáticas em app/database.py → run_migrations(), executadas no startup da aplicação. Não usa Alembic.

Quando uma coluna nova é adicionada ao modelo, uma entrada é adicionada em run_migrations():

if "penalty_winner" not in existing_cols:
    pending.append("ALTER TABLE matches ADD COLUMN penalty_winner VARCHAR")

As colunas existentes atualmente migradas automaticamente incluem: prediction_deadline, penalty_winner, reset_token, reset_token_expires e outros campos adicionados ao longo do desenvolvimento.

Força recálculo do ranking geral.

Lógica de Cálculo

  1. Ao finalizar uma partida via PUT /api/matches/{id}/score, os pontos são calculados para cada palpite.
  2. O ranking da rodada é recalculado automaticamente (soma dos pontos de todas as partidas da rodada).
  3. O ranking geral é recalculado (soma de todos os pontos históricos do usuário).
  4. Em caso de empate: desempate por número de acertos exatos.

Prevenção de Duplicatas (Multi-worker)

Em produção com Gunicorn (3 workers), o scheduler usa dois mecanismos:

  1. File lock (/tmp/bolao_scheduler.lock) — apenas um worker executa o scheduler via fcntl.flock().
  2. UPDATE atômico — antes de enviar, executa UPDATE scheduled_messages SET status='sending' WHERE id=:id AND status IN ('scheduled','pending'). Se rowcount=0, outro worker já processou: pula.

Serviço — Calculadora de Pontos

Arquivo: app/services/points_calculator.py. Função principal: calculate_points(match, prediction) → (total, winner, score_a, score_b, exact).

def calculate_points(match, prediction):
    winner = 2 if acertou_vencedor_ou_empate else 0
    score_a = 2 if pred.predicted_score_a == match.score_a else 0
    score_b = 2 if pred.predicted_score_b == match.score_b else 0
    exact = 2 if score_a and score_b else 0
    total = winner + score_a + score_b + exact
    return total, winner, score_a, score_b, exact

Chamada automaticamente ao finalizar cada partida. O resultado é salvo nos campos points_* do model Prediction.

Serviço — Classificação

Arquivo: app/services/standings_service.py. Atualiza grupos e mata-mata automaticamente.

Funções principais

  • update_group_standings(db, group) — recalcula a tabela do grupo a partir de todos os jogos finalizados.
  • check_and_update_knockout_matches(db) — quando todos os grupos são decididos, preenche as partidas dos 16 avos com 1ºs, 2ºs e melhores 3ºs.
  • update_knockout_match(db, match_number, winner, winner_code) — após um jogo do mata-mata, procura no DB qual partida futura tem "Vencedor {N}" ou "W{N}" no nome do time e atualiza com o vencedor real.
  • auto_update_standings_on_match_finish(db, match) — função central chamada ao finalizar qualquer jogo. Decide automaticamente o que atualizar.
Empate no mata-mata: se score_a == score_b e penalty_winner for null, o sistema aguarda e não avança nenhum time. O admin deve preencher penalty_winner ("A" ou "B") ao finalizar a partida.
Bracket errado? Use Admin → Configurar Fase Eliminatória (/admin/16avos) para corrigir manualmente o nome e código dos times em qualquer fase (16 avos, oitavas, quartas, semis, 3º lugar, final).

Guia do Usuário

1. Cadastro

  1. Acesse /cadastro e preencha nome, e-mail e WhatsApp.
  2. Confirme o código recebido no e-mail.
  3. Opcionalmente confirme também o WhatsApp.
  4. Aguarde o admin confirmar seu pagamento de inscrição (R$ 100).

2. Fazer Palpites

  1. Acesse /palpites.
  2. Selecione a rodada desejada (só aparece se o pagamento da rodada estiver confirmado).
  3. Para cada jogo, insira o placar que acha que vai acontecer nos 90 minutos.
  4. Clique em Salvar Palpite. Você pode alterar enquanto o prazo estiver aberto.
  5. O prazo fecha automaticamente 10 minutos antes de cada partida.

3. Acompanhar Partidas

  • /partidas — ao vivo, jogos de hoje, próximos jogos, últimos resultados.
  • /grupos — classificação dos 12 grupos da Copa 2026.

4. Rankings

  • /ranking — selecione uma rodada ou veja o ranking geral.
  • Pontuação máxima: 8 pontos por jogo.
  • Desempate: maior número de placares exatos.

5. PVP — Apostas Diretas

  • Acesse /pvp para desafiar outros usuários em jogos específicos.
  • Defina o valor apostado e quem você acha que vai ganhar.
  • O adversário aceita ou recusa. O resultado é determinado quando o jogo finalizar.

6. Transparência

  • Acesse /transparencia para ver o histórico financeiro do bolão.

  3. Faça login — o menu admin aparecerá automaticamente.

Popular Partidas

  1. Acesse /admin → Ações Rápidas → Popular banco de dados (ou chame POST /api/admin/matches/seed).
  2. Todas as 104 partidas da Copa 2026 serão criadas com datas, horários, cidades e estádios.

  1. Acesse Central de Cobrança (/admin/cobrancas).
  2. Veja todos os usuários e o status de inscrição + cada rodada.
  3. Clique no campo correspondente para confirmar ou registrar o pagamento.

  1. Acesse /admin → Gerenciar Jogos.
  2. Selecione o jogo no dropdown.
  3. Insira o placar dos 90 minutos.
  4. Se for mata-mata e houve empate, selecione quem avançou (pênaltis/prorrogação).
  5. Clique em Finalizar Jogo. Pontos e rankings são calculados automaticamente.

  1. Acesse /admin/16avos.
  2. Selecione a fase (16 Avos, Oitavas, Quartas, Semis, 3º Lugar, Final) nos botões de fase.

  4. Clique em Salvar Partida.

  1. Acesse Central de Mensagens (/admin/mensagens).
  2. Crie uma mensagem com título, corpo, canal (email/WhatsApp/ambos) e destinatários (todos/pagantes/usuário específico).
  3. Escolha enviar imediatamente ou agendar para data/hora específica.
  4. Mensagens recorrentes (diário/semanal) também são suportadas.

Configurar WhatsApp

  2. Acesse /admin/whatsapp e configure a URL, instância e API Key.

Fase Eliminatória — Como Funciona

Preenchimento Automático do Bracket

Ao finalizar um jogo da fase de grupos, o sistema verifica se todos os grupos estão definidos. Quando sim, preenche automaticamente os 16 avos de final baseado nos seguintes placeholders no seed data:

  • team_a = "1º Grupo A", team_a_code = "1A" → substituído pelo 1º do Grupo A
  • team_a = "2º Grupo B", team_a_code = "2B" → substituído pelo 2º do Grupo B

Ao finalizar um jogo do mata-mata (16avos, oitavas, etc.), o sistema procura a próxima partida que contenha "Vencedor {N}" ou "W{N}" no nome do time e substitui pelo vencedor real.

Quando o Bracket Fica Errado

Isso acontece quando:

  • A FIFA alterou os confrontos após o seed original
  • Os placeholders no DB não batem com o padrão esperado pelo algoritmo
  • O admin corrigiu os 16 avos manualmente, mas as oitavas ainda tinham placeholder antigo

Solução: use /admin/16avos para corrigir manualmente qualquer fase. O sistema de palpites funciona com o nome que estiver no banco — o que importa é que o time correto apareça antes do prazo de palpite.

Regras do Mata-mata

  • Pontuação dos palpites: somente pelo placar dos 90 minutos
  • Prorrogação: não conta para pontos, mas pode ser o resultado real
  • Pênaltis: o campo penalty_winner define quem avança de fase (sem afetar pontuação)
  • Admin deve preencher penalty_winner ao finalizar jogos empatados

Fluxo Completo de uma Partida

  → PUT /api/matches/{id}/score com score + status=finished
  → calculate_points() para cada palpite

     → Fase de grupos: update_group_standings() + check_and_update_knockout_matches()
     → Mata-mata: update_knockout_match() (preenche próxima fase)
  → calculate_round_ranking_internal()
  → calculate_general_ranking_internal()
